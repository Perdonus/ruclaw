# Навык создания навыков

Этот навык помогает создавать хорошие навыки без лишнего мусора в контексте.

## Что такое навык

Навык — это модульный пакет, который расширяет возможности агента:
- добавляет специализированный процесс;
- описывает работу с инструментом, API или форматом;
- добавляет папки scripts, references и assets для повторяемых задач.

## Главные принципы

### 1. Пишите кратко

Контекст — общий ресурс. Навык делит его с системной инструкцией, историей, другими навыками и самим запросом пользователя.

Используйте только ту информацию, которую модель сама не выведет надёжно.

### 2. Выбирайте правильную степень свободы

- **Высокая**: текстовые инструкции и эвристики, когда вариантов много.
- **Средняя**: псевдокод или параметризуемые скрипты, когда есть предпочтительный шаблон.
- **Низкая**: конкретные команды и жёсткие последовательности, когда ошибка дорого стоит.

### 3. Делайте постепенное раскрытие

У навыка три уровня загрузки:
1. метаданные (`name`, `description`);
2. тело `SKILL.md`;
3. подключаемые ресурсы по мере необходимости.

Храните в `SKILL.md` только ядро процесса. Детали выносите в `references/`, код — в `scripts/`, материалы вывода — в `assets/`.

## Структура навыка

```
skill-name/
├── SKILL.md
├── scripts/      # исполняемый код
├── references/   # документация и справки
└── assets/       # шаблоны, иконки, шрифты и т.п.
```

### Что обязательно должно быть в SKILL.md

- YAML frontmatter с `name` и `description`
- Чёткое описание: что делает навык и когда его надо включать
- Инструкции по процессу
- Ссылки на `references/` при необходимости

### Чего делать не надо

Не создавайте для навыка лишнюю обвязку вроде:
- `README.md`
- `CHANGELOG.md`
- `INSTALLATION_GUIDE.md`
- `QUICK_REFERENCE.md`

Если файл не помогает агенту выполнять задачу, ему не место в навыке.

## Процесс создания навыка

1. Понять задачу и реальные примеры использования
2. Выделить повторяемые части: процесс, scripts, references, assets
3. Создать каркас навыка
4. Написать `SKILL.md` и приложить ресурсы
5. Проверить навык на реальном сценарии
6. Упростить всё, что оказалось лишним

## Правила именования

- Только lowercase, цифры и дефисы
- Имя должно отражать действие
- Папка навыка должна называться ровно как навык
- По возможности делайте название коротким и ясным

## Какие вопросы задавать пользователю

Если нужно уточнение, задавайте вопросы по одному приоритетному блоку:
- какой результат должен получить агент;
- что должно быть детерминированным;
- какие инструменты и форматы уже используются;
- нужны ли references, scripts или assets.

Не задавайте сразу длинную простыню вопросов. Лучше 1-2 приоритетных вопроса и затем следующая итерация.
