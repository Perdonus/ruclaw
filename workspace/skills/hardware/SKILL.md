# Навык работы с I2C и SPI

Используйте инструменты `i2c` и `spi` для работы с датчиками, дисплеями и другой периферией, подключённой к плате.

## Быстрый старт

```
# 1. Найти доступные шины
i2c detect

# 2. Просканировать устройства
i2c scan  (bus: "1")

# 3. Прочитать датчик
i2c read  (bus: "1", address: 0x38, register: 0xAC, length: 6)

# 4. SPI-устройства
spi list
spi read  (device: "2.0", length: 4)
```

## Перед началом: pinmux

На многих платах Sipeed линии I2C/SPI разделяют пины с Wi‑Fi. Перед работой настройте pinmux.

См. `references/board-pinout.md` для команд под конкретную плату.

Общий порядок:
1. Остановить Wi‑Fi, если используются общие пины: `/etc/init.d/S30wifi stop`
2. Загрузить модуль `i2c-dev`: `modprobe i2c-dev`
3. Настроить pinmux через `devmem` или `pinmap`
4. Проверить результат через `i2c detect` и `i2c scan`

## Безопасность

- **Операции записи** требуют `confirm: true` — всегда сначала уточняйте у пользователя
- I2C-адреса проверяются в диапазоне 7-bit (0x03-0x77)
- SPI-режимы ограничены значениями 0-3
- Максимум за одну транзакцию: 256 байт (I2C), 4096 байт (SPI)

## Типичные проблемы

| Проблема | Решение |
|---------|---------|
| Не найдены I2C-шины | Выполнить `modprobe i2c-dev` и проверить device tree |
| Отказано в доступе | Запускать от root или добавить пользователя в группу `i2c` |
| Скан не видит устройств | Проверить проводку, подтяжки и pinmux |
| Номер шины изменился | Перед запуском заново вызвать `i2c detect` |
| Перестал работать Wi‑Fi | Некоторые I2C/SPI-линии конфликтуют с Wi‑Fi SDIO |
| Не найден `devmem` | Использовать `busybox devmem` или поставить утилиту отдельно |
| SPI возвращает нули | Проверить питание устройства и линию MISO |
| SPI возвращает 0xFF | Устройство не отвечает; проверить CS и SPI mode |
