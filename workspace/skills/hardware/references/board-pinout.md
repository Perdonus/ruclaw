# Распиновка плат и справка по pinmux

## LicheeRV Nano (SG2002)

### I2C-шины

| Шина | Пины | Примечание |
|------|------|------------|
| I2C-1 | P18 (SCL), P21 (SDA) | **Общие с Wi‑Fi SDIO** — сначала нужно остановить Wi‑Fi |
| I2C-3 | Доступна на гребёнке | Проверьте device tree для точного назначения пинов |
| I2C-5 | Программная (BitBang) | Медленнее, но без конфликта по пинам |

### SPI-шины

| Шина | Пины | Примечание |
|------|------|------------|
| SPI-2 | P18 (CS), P21 (MISO), P22 (MOSI), P23 (SCK) | **Общая с Wi‑Fi** — сначала остановите Wi‑Fi |
| SPI-4 | Программная (BitBang) | Медленнее, но без конфликта |

### Настройка I2C-1

```bash
# 1. Остановить Wi‑Fi
/etc/init.d/S30wifi stop

# 2. Настроить pinmux для I2C-1
devmem 0x030010D0 b 0x2   # P18 → I2C1_SCL
devmem 0x030010DC b 0x2   # P21 → I2C1_SDA

# 3. Загрузить i2c-dev
modprobe i2c-dev

# 4. Проверить
ls /dev/i2c-*
```

### Настройка SPI-2

```bash
# 1. Остановить Wi‑Fi
/etc/init.d/S30wifi stop

# 2. Настроить pinmux для SPI-2
devmem 0x030010D0 b 0x1   # P18 → SPI2_CS
devmem 0x030010DC b 0x1   # P21 → SPI2_MISO
devmem 0x030010E0 b 0x1   # P22 → SPI2_MOSI
devmem 0x030010E4 b 0x1   # P23 → SPI2_SCK

# 3. Проверить
ls /dev/spidev*
```

### Максимальная проверенная скорость SPI

- Аппаратный SPI-2: до **93 МГц**
- `spidev_test` предустановлен в официальном образе для loopback-проверок

## MaixCAM

### I2C-шины

| Шина | Пины | Примечание |
|------|------|------------|
| I2C-1 | Пересекается с Wi‑Fi | Не рекомендуется |
| I2C-3 | Пересекается с Wi‑Fi | Не рекомендуется |
| I2C-5 | A15 (SCL), A27 (SDA) | **Рекомендуется** — программная I2C, без конфликта |

### Настройка I2C-5

```bash
# Настройка pinmap
# (MaixCAM использует pinmap вместо devmem)
# См.: https://wiki.sipeed.com/hardware/en/maixcam/gpio.html

modprobe i2c-dev
ls /dev/i2c-*
```

## Типичные проблемы

### Не найден devmem

- Попробуйте `busybox devmem`
- Установите утилиту отдельно
- Или соберите её из исходников

### Динамическая нумерация шин

Номера I2C-адаптеров могут меняться между перезагрузками. Перед каждым запуском лучше снова вызвать `i2c detect`.

### Права доступа

`/dev/i2c-*` и `/dev/spidev*` обычно требуют root-доступа. Варианты:
- Запустить RuClaw от root
- Добавить пользователя в группы `i2c` и `spi`
- Создать udev-правила, например `SUBSYSTEM=="i2c-dev", MODE="0666"`
